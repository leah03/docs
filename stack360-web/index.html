Features

Automate deployment of data center infrastructure and services from anywhwere.
Stack360 helps you create applications on public cloud services such as AWS and Microsoft Azure,
or on OpenStack cluster in your private cloud. It can stand up data center infrastructure, if needed.

Managed Infra Cloud

Infrastructure and service as configuration file

Stack360 Managed Infra Cloud Services(MICS) allow you to implement Infrastructure as Code practice, with infrastructure and application resources represented as blueprints. MICS automate the provisioning of these resources.

Recomposable

Recomposable hardware

With our automation and workflow support tools, you can easily create, de-commission, and re-create stacks with your data center hardware resource pools.

Remote Control

Don't have to be onsite

With Stack360's web-based service portal, you can control the data center configuration from anywhere. You can also choose to monitor your infrastructure (such as alarms, and log files) from our servies.

Getting started

Download local Agent

First, download the agent source code from github to your local machine.

git clone -b git://git.openstack.org/openstack/compass-core.git  
cd compass-core

Configure agent environment

There are a few settings to be configured so that the agent is aware of local environment.
In /compass-core/install/ folder, open the file install.conf

export USER_EMAIL="lab@huawei.com
export PUBLIC_NIC="eth0
export NIC="eth1

put your user email USER_EMAIL
put the physical nic that connects to the public network PUBLIC_NIC
put the install nic NIC
NOTE: if you have configured the install nic's IP address and netmask, you should leave those variables unassigned. If you have an unconfigured install NIC, please assign the values to IPADDR and NETMASK and install.sh will configure the install interface for you.

Install the agent

Run the install script to install the agent

./install/install.sh

Register User

Your user account has to be activated and registered to Stack360 server. Run the register script to register.

./install/register.sh

This script will return a password for the registered user account. Please store this password for future uses.

You can start configure your first cluster!


Login in to Stack360 UI stack360.io with your credential

Note: Your account has to be activated(step 4 only registers your account). If your account has not been activated, you need to contact us to have your account activated

About

The remote deployment service is part of the Stack360 project.
Stack360 is an open-source tool that helps users to configure, deploy and manage their data center from baremetals to application orchestration.


View Stack360 Project

How it works

Deployment automation engine is provided by Stack360, an open source project.
It supports OpenStack deployment, blueprint-based application orchestration in a multi-cloud environment.
Stack360 MICS adds deployment as a service capabilities by adding version control, multi-tenant,
remte management, and other features.

Install

Install a local agent in your target environment- whether data center or cloud.

Submit

Users submit desired configuration files to stack360 server.

Deploy

The stack360 server will go ahead deploying infrastructure and services.
